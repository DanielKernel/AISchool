# AI 算法一日速通指南（合并版）

> **目标**：在一天内掌握本仓库中的 **10 个** 高频考试算法：能默写核心循环、说清原理与易错点，并通过闭卷或上机考核。  
> **代码**：所有算法的**唯一实现**在 [`aischool/algorithm_core.py`](aischool/algorithm_core.py)；[`algorithms/*.py`](algorithms/) 为带详解的薄封装；[`examples/algorithms_reference.py`](examples/algorithms_reference.py) 提供自检；根目录 [`machine_learning.py`](machine_learning.py)、[`graph_algorithms.py`](graph_algorithms.py)、[`dl_basics.py`](dl_basics.py) 为分主题演示；[`run_all.py`](run_all.py) 一键跑通演示。

依赖：`pip install numpy`（见 [`requirements.txt`](requirements.txt)）。

---

## 一、学习总策略

```
理解原理 → 看代码 → 手写 → 运行验证 → 总结口诀
```

每个算法只需掌握：

1. **一句话原理**（能说清楚在做什么）
2. **核心数据结构**（用什么存中间状态）
3. **关键循环/递推**（约 5～15 行骨架）
4. **终止条件**（何时停止）

**一天内要达成的可检验目标**（与机试/笔试对齐）：

- 能**不看资料**写出每个算法的核心循环或关键公式对应的代码骨架。
- 能说明**时间复杂度**、**输入输出**、**1～2 个易错点**（如 K-Means 初值敏感、PageRank 阻尼因子、悬挂节点）。
- 能在 **15～30 分钟** 内复现一个算法的「最小正确版本」。

## 二、如何加强练习（推荐闭环）

以下方法在一天内可反复多轮，比单纯「多读几遍」更有效。

### 1. 主动回忆（Active Recall）

- 每学完一个算法，**合上所有材料**，在纸上只写：函数签名、主循环、2～3 个关键赋值行。
- 写完后**立即**对照 `algorithm_core.py` 或对应 `algorithms/0X_*.py` 的 `if __name__` 示例，用红笔标出遗漏（尤其是索引、归一化、`1 - color` 这类细节）。

### 2. 间隔重复（同一天内模拟）

- **上午块**学完算法 1～5 后，**午饭前**不看笔记默写其中任意 2 个的骨架。  
- **下午块**学完 6～10 后，**晚饭前**再默写上午的 2 个 + 下午任选 2 个（共 4 个，约 40 分钟）。  
- **睡前 15 分钟**：只过一遍 **10 个名字 + 各一行核心公式**（记忆巩固）。

### 3. 交错练习（Interleaving）

- 不要连续 3 小时只练卷积；改为 **K-Means → PageRank → 卷积** 这类跨主题交替，更接近考试「题目乱序」的真实负荷。
- 用 [`run_all.py`](run_all.py) 与 `python examples/algorithms_reference.py` **交错运行**：前者看演示输出，后者用 assert 压实边界条件。

### 4. 限时模拟考（闭卷）

- 设 **60～90 分钟** 闹钟，从空文件开始写 **10 个函数名 + 各自主循环**（变量名可简写，但循环边界与公式不能错）。
- 结束后用仓库代码 **只查漏、不重抄整文件**，把错因记成一句「我下次先检查 ___」。

### 5. 口述与白板

- 对每个算法用 **60 秒** 讲给假想的同学听：输入是什么、输出是什么、为什么这一步要这样做（例如：SMOTE 为何在连线上插值、Kruskal 为何要用并查集）。

### 6. 易错点清单（考前 30 分钟只看这一页）

- **K-Means vs KNN**：无监督簇数 K vs 监督近邻数 K。  
- **熵公式**：`H = -Σ p_i log2(p_i)`，约定 `0*log(0)=0`。  
- **PageRank**：阻尼常用 0.85；邻接矩阵版注意行归一化与转置；悬挂节点勿除零。  
- **Kruskal**：并查集 **find 路径压缩**、**union 按秩**；边数到 **n−1** 停止。  
- **卷积/池化输出尺寸**：`(N - K + 2P) / S + 1`（整除用 `//`）。  
- **SMOTE**：新样本在 **两点连线上**，近邻索引勿把自身当邻居。

## 四、算法详解与记忆口诀

### 1. K-Means（聚类）

**一句话原理**：随机初始化 K 个中心点，反复将每个样本分配给最近的中心，再更新中心为所属簇的均值，直到中心不再变化。

**口诀**：`随机选中心 → 分配最近簇 → 更新均值 → 重复直到收敛`

**关键骨架**：

```python
for _ in range(max_iter):
    labels = [argmin(dist(x, center)) for x in X]
    new_centers = [mean(X[labels==k]) for k in range(K)]
    if 收敛: break
    centers = new_centers
```

**考点**：欧氏距离；空簇时保留旧心；终止条件。

---

### 2. KNN（分类）

**一句话原理**：预测新样本时，找训练集中欧氏距离最近的 K 个邻居，投票（分类）或取均值（回归）。

**口诀**：`算距离 → 排序取 K 近 → 投票/均值`

**考点**：lazy learning；K 为偶数平票需约定；量纲敏感时可标准化。

---

### 3. PageRank（网页排名）

**一句话原理**：排名 = 入链贡献（按出度归一）+ 随机跳转；幂迭代至收敛。

**口诀**：`PR = d * 入链贡献之和 + (1-d)/N`

**考点**：阻尼 d≈0.85；悬挂节点；初值常取 1/N。

---

### 4. 决策树——信息增益

**一句话原理**：选使信息增益最大的特征分裂。信息增益 = 父节点熵 − 子节点加权熵之和。

**口诀**：`熵(父) - Σ(子比例 × 熵(子))` → 取最大

**考点**：ID3 / C4.5 / CART 区别；`log2` 与 `log` 别混。

---

### 5. 卷积（滑动窗口）

**一句话原理**：卷积核在输入上滑动，重叠区域逐元素乘再求和。

**口诀**：`输出[i][j] = sum(输入窗口 * 卷积核)`

**考点**：步长 stride、padding；输出尺寸公式。

---

### 6. Max Pooling

**一句话原理**：窗口内取最大值，降采样。

**口诀**：`输出[i][j] = max(窗口)`

**考点**：与卷积同属滑动窗口框架；通常无可学习参数。

---

### 7. 二分图判断（BFS 染色）

**一句话原理**：BFS 交替染色 0/1；相邻同色则非二分图。

**口诀**：`BFS 染色 → 相邻异色 → 冲突即非二分图`

**考点**：非连通图需对每个连通分量各做一次。

---

### 8. Kruskal（最小生成树）

**一句话原理**：边按权排序，贪心加边，并查集判环，直到 n−1 条边。

**口诀**：`排边权 → 贪心加边 → 并查集判环`

**考点**：路径压缩与按秩合并；与 Prim 的适用场景对比。

---

### 9. 梯度下降（线性回归）

**一句话原理**：沿损失对参数的梯度反方向更新，重复至收敛。

**口诀**：`预测 → 误差 → 梯度 → w -= lr * grad`

**考点**：MSE；学习率过大震荡；BGD / SGD 概念。

---

### 10. SMOTE（过采样）

**一句话原理**：少数类样本与其 K 近邻连线上随机插值生成新样本。

**口诀**：`少数类 → K近邻 → 随机插值`

**考点**：`new = x + λ*(xn - x)`，λ∈[0,1]；勿在测试集上做 SMOTE。

## 五、考前速查：算法对比表

| 算法 | 类别 | 核心操作 | 关键参数 |
|------|------|---------|---------|
| K-Means | 无监督聚类 | 欧氏距离 + 均值更新 | K |
| KNN | 监督分类 | 欧氏距离 + 投票 | K |
| PageRank | 图排名 | 幂迭代 | d≈0.85 |
| 决策树 | 监督分类 | 信息增益/熵 | 深度等 |
| 卷积 | 特征提取 | 滑动窗口乘加 | 核、步长、padding |
| Max Pooling | 降采样 | 滑动窗口取最大 | 窗口、步长 |
| 二分图 | 图判断 | BFS 染色 | — |
| Kruskal | MST | 贪心 + 并查集 | — |
| 梯度下降 | 优化 | 梯度更新 | lr |
| SMOTE | 不平衡 | K 近邻 + 插值 | K |

## 六、默写练习方法（最小流程）

1. **看一遍代码**（约 2 分钟）：只看循环与关键变量。  
2. **合上手写**（约 5 分钟）：只写核心逻辑。  
3. **对比查漏**（约 2 分钟）。  
4. **再手写一遍**（约 3 分钟）：不看任何提示。

每个算法重复 **2～3 轮**，一般即可记牢。
